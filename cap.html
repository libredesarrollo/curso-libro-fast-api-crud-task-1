Capítulo: Interfaz de Usuario con Jinja2, Archivos Estáticos y un Portal Central

Hasta ahora hemos construido una API sólida con FastAPI: rutas para gestionar tareas, usuarios, autenticación mediante tokens y carga de archivos. Sin embargo, toda esa funcionalidad existe de forma aislada. Quien visite nuestra aplicación en el navegador no tiene una forma clara de saber qué puede hacer, ni cómo navegar entre las diferentes secciones.

En este capítulo vamos a resolver eso. Crearemos una página central que sirva como punto de entrada a la aplicación, conectará visualmente todos los módulos que ya tenemos implementados y aprovechará las plantillas Jinja2 para renderizar HTML desde el servidor. También aprenderemos a servir archivos estáticos (CSS, imágenes, JavaScript) desde FastAPI.

Al final del capítulo tendremos una aplicación con navegación completa, autenticación funcional desde el navegador, carga de archivos con arrastrar y soltar, y pruebas de endpoints protegidos, todo sin salir de la interfaz web.

2. Servir archivos estáticos: api.py

FastAPI, a través de Starlette, permite montar directorios de archivos estáticos en una ruta específica. Esto es indispensable para servir hojas de estilo CSS, imágenes y cualquier otro recurso que el navegador deba descargar directamente.

Importamos StaticFiles desde fastapi.staticfiles y usamos el método mount sobre nuestra instancia de la aplicación. También añadimos la ruta raíz GET / que renderizará nuestro portal central: 

# api.py (fragmento)

from fastapi import FastAPI, Depends, APIRouter, Query, Path, Request, Header, HTTPException, status
from fastapi.security import APIKeyHeader
from fastapi.templating import Jinja2Templates
from fastapi.staticfiles import StaticFiles

from typing import Optional
from typing_extensions import Annotated

from sqlalchemy.orm import Session

templates = Jinja2Templates(directory="templates/")

# uvicorn api:app --reload

app = FastAPI()
app.mount("/static", StaticFiles(directory="static"), name="static")

router = APIRouter()

Base.metadata.create_all(bind=engine)

@app.get('/')
def home(request: Request):
    return templates.TemplateResponse(request=request, name='home.html')

Observa dos puntos importantes:

  1. El directorio static/ debe existir antes de iniciar el servidor. Si no existe, FastAPI lanzará un error al intentar montarlo.
  2. Usamos la forma moderna de llamar a TemplateResponse: el argumento request se pasa como keyword argument en lugar de incluirlo dentro del diccionario de contexto. Esta es la firma recomendada desde las versiones recientes de Starlette. Si usas la forma antigua con una versión nueva, obtendrás un TypeError: unhashable type: 'dict'.

La ruta /page que renderiza la lista de tareas también se actualiza con la misma firma moderna, pasando los datos de contexto (tareas y categorías) en el parámetro context: 

# api.py (fragmento)

@app.get('/page')
def index(request: Request, db: Session = Depends(get_database_session)):
    categories = db.query(Category).all()
    return templates.TemplateResponse(
        request=request,
        name='task/index.html',
        context={'tasks': crud.getAll(db), 'categories': categories}
    )

Con estos cambios en api.py, el servidor ya sabe cómo servir archivos estáticos desde /static/... y cómo renderizar las plantillas HTML en las rutas / y /page.

El comando para iniciar el servidor con recarga automática es: 

uvicorn api:app --reload

3. El sistema de diseño: static/css/style.css

Creamos una hoja de estilos CSS completa que define el aspecto visual de toda la aplicación. Utilizamos variables CSS (custom properties) para centralizar los valores de color, tipografía y espaciado, lo que facilita modificar el tema en un solo lugar.

El sistema de diseño adopta un tema oscuro (dark mode) con tarjetas glasmórficas, usando backdrop-filter: blur para el efecto de cristal esmerilado. La tipografía principal es Outfit, cargada desde Google Fonts.

        function globalLogout() {
            const token = localStorage.getItem('access_token');
            if (!token) return;
            fetch('/logout', {
                method: 'DELETE',
                headers: { 'Token': token }

                localStorage.removeItem('access_token');
                window.showToast('Sesión cerrada con éxito', 'success');
                updateAuthUI();
                window.dispatchEvent(new Event('authChange'));
            })
            .catch(err => {
                localStorage.removeItem('access_token');
                updateAuthUI();
            });
        }

        updateAuthUI();
        window.addEventListener('authChange', updateAuthUI);

    function switchTab(tabId, event) {
        document.querySelectorAll('.tab-btn').forEach(b => b.classList.remove('active'));
        document.querySelectorAll('.tab-content').forEach(c => c.classList.remove('active'));
        event.currentTarget.classList.add('active');

    // --- Login: obtiene token desde POST /token ---
    function handleLogin(e) {
        e.preventDefault();
        const form = e.target;
        const formData = new URLSearchParams();
        formData.append('username', form.username.value);
        formData.append('password', form.password.value);

        fetch('/token', {
            method: 'POST',
            headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
            body: formData
        })
        .then(res => {
            if (!res.ok) throw new Error('Credenciales inválidas');
            return res.json();
        })
        .then(data => {
            localStorage.setItem('access_token', data.access_token);
            window.showToast('Acceso autorizado. Token guardado.', 'success');
            window.dispatchEvent(new Event('authChange'));
            form.reset();
        })
        .catch(err => window.showToast(err.message, 'danger'));
    }

    // --- Registro: crea usuario en POST /register ---
    function handleRegister(e) {
        e.preventDefault();
        const form = e.target;
        const data = {
            name: form.name.value,
            surname: form.surname.value,
            email: form.email.value,
            website: form.website.value,
            password: form.password.value

        fetch('/register', {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify(data)
        })
        .then(async res => {
            const resData = await res.json();
            if (!res.ok) throw new Error(resData.detail || 'Error al registrar');
            return resData;
        })
        .then(data => {
            window.showToast(data.message || 'Registro exitoso.', 'success');
            document.querySelector('#tab-login input[name="username"]').value = form.email.value;
            form.reset();
        })
        .catch(err => window.showToast(err.message, 'danger'));

<script>
    function switchCreateMode(mode) {
        const isAjax = mode === 'ajax';
        document.getElementById('btn-create-ajax').classList.toggle('active', isAjax);
        document.getElementById('btn-create-form').classList.toggle('active', !isAjax);
        document.getElementById('create-mode-ajax').style.display = isAjax ? 'block' : 'none';
        document.getElementById('create-mode-form').style.display = isAjax ? 'none' : 'block';
    }

    document.getElementById('createTask').addEventListener('click', function () {
        const f = document.getElementById('createForm');
        const name      = f.querySelector('input[name="name"]').value;
        const category  = f.querySelector('select[name="category_id"]').value;

        if (!name || !category) {
            window.showToast('Nombre y categoría son requeridos', 'warning');
            return;
        }

        const data = {
            name,
            description: f.querySelector('textarea[name="description"]').value,
            status:      f.querySelector('select[name="status"]').value,
            category_id: parseInt(category),
            user_id:     parseInt(f.querySelector('input[name="user_id"]').value)
        };

        fetch('/tasks/', {
            method: 'POST',
            body: JSON.stringify(data),
            headers: { 'Content-type': 'application/json' }
        })
        .then(res => {
            if (!res.ok) return res.json().then(e => { throw new Error(e.detail); });
            return res.json();
        })
        .then(() => {
            window.showToast('¡Tarea creada con éxito!', 'success');
            setTimeout(() => window.location.reload(), 1000);
        })
        .catch(err => window.showToast(err.message, 'danger'));
    });
</script>

9. Cómo iniciar el servidor

El archivo principal de la aplicación se llama api.py. El comando de Uvicorn debe apuntar exactamente a ese nombre de archivo (sin extensión) seguido de :app, que es el nombre del objeto FastAPI definido dentro: 

# Activa el entorno virtual primero (si usas uno)
source .venv/bin/activate

# Luego inicia el servidor con recarga automática
uvicorn api:app --reload

Un error frecuente es escribir un nombre de archivo incorrecto, por ejemplo uvicorn archivo:app --reload. Uvicorn intentará importar un módulo llamado archivo y fallará con el mensaje Could not import module "archivo". El nombre que se le pasa a Uvicorn debe coincidir exactamente con el nombre del archivo Python sin la extensión .py.
